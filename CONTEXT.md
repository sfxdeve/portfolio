# Portfolio

Personal developer portfolio for Shayan Fareed: projects presented as craft case studies on a showcase-heavy craft-logbook site. v1 ships with two to three Case Studies.

## Language

**Case Study**:
A project page with a Capsule meta-rail and a body of alternating short narrative blocks and Showcase sections. Proof lives in the artifacts; text explains problem, constraints, and outcome—not chapters.
_Avoid_: Essay, article, long-form writeup, blog post, showcase-only gallery, rigid multi-section template without visual proof, single-column Capsule grid as the v1 Case Study layout

**Capsule**:
The scannable Case Study summary—problem, role, stack, and outcome—shown in a sticky/side meta-rail on wider viewports so a hiring-first reader can skim while reading the alternating body. Technical depth follows for peers without being required to get the point.
_Avoid_: Manifesto intro, long abstract, pull-quote opener, peer-only jargon in the first screen, Capsule as a long top-of-page essay block
**Showcase**:
Visual or interactive proof of the work—product UI, recordings, diagrams, live demos, code diffs—used as a first-class part of a Case Study, not decoration around a wall of text. Default is artifact-first (screens, recordings, diagrams, diffs); interactive demos are optional and reserved for flagship cases that need them.
_Avoid_: Hero illustration, stock photography, abstract gradient mesh as the main visual, demo-obligatory portfolio

**Craft Logbook**:
The site’s overall direction: a personal index of work that opens into Case Studies, signaling the site was built by a developer through restraint, systems thinking, and interaction craft—not agency marketing or editorial magazine branding. Default presentation is light, neutral surfaces with a low-chroma teal/blue-green accent used for meaning (links, focus, interactive states)—not identity washes. Typography is Geist Sans for UI/prose and Geist Mono for meta, code, and spec-like labels. Motion is sparse UI feedback and purposeful Showcase demos; no scroll theatre. Shared/view transitions are optional later, not part of the initial identity.
_Avoid_: Editorial, magazine layout, designer-for-hire folio, SaaS marketing landing page, dark purple-glow “dev cosplay”, display serif heroes, gradient brand themes, terracotta-on-cream, parallax/scroll-hijack heroes

**Home**:
The landing experience: a short identity strip (name + “product engineer” role) with the work index beginning in the first viewport as dense numbered list rows (title, one-line outcome, year/stack meta)—not cards. Longer bio lives on a separate thin About surface if needed. v1 surfaces are Home, Case Studies, and thin About; contact is links in chrome, not a Writing section or dedicated contact page. Layout reference: prototype Variant B (meta-rail Case Study).
_Avoid_: Work-only home, manifesto hero, cryptic craft-only landing, tall about-me first screen, card-grid project gallery, blog/writing index, contact-form page as a v1 requirement, “design engineer” as the primary Home label
