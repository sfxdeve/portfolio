# Use TanStack Start With MDX Detail Routes For An Evidence-Led Portfolio

We will build the portfolio as a Vercel-friendly TanStack Start app where the homepage is a React-composed narrative sequence, MDX owns sourced content for shipped work and explorations, and detail routes provide case-study evidence. Each MDX file owns structured frontmatter for homepage summaries and metadata alongside the narrative body for its detail route, with no separate summary registry. This favors an interactive, evidence-led product experience over a simpler static resume site, while keeping authored evidence maintainable and separating the portfolio's implementation choices from the technical range proven by delivered systems.

## Considered Options

- A static one-page portfolio would be simpler to build but would make detailed case-study evidence, inspectable interactions, and maintainable shipped-product narratives weaker.
