# Craft logbook visual direction (showcase-heavy case studies)

We are building a product-engineer portfolio as a **showcase-heavy Craft Logbook**: Home is a short identity strip plus a dense work-index list in the first viewport; each **Case Study** uses a sticky/side **Capsule** meta-rail with an alternating short-text + Showcase body—not essays, not editorial magazine layouts, not a designer-for-hire folio.

**Why this over the alternatives:** Editorial was tried and rejected. Long-form essay / interactive-notebook directions (Heckel/Comeau-style) over-weight writing. A personal app-shell (Lovin-style) is a strong navigation model but secondary to craft-logbook tone for v1. Dark-as-identity and purple-glow SaaS aesthetics were rejected as costume. Light neutrals, Geist Sans + Geist Mono, a low-chroma teal accent for meaning only, sparse UI motion, and artifact-first Showcases keep the site reading as developer-built.

**Prototype verdict (2026-07-16):** Variant B (meta-rail Case Study) won over single-column Capsule (A) and showcase-leads-each-block (C). Throwaway primary source archived on branch `prototype/craft-logbook-ui` (route `/prototype/craft-logbook?variant=B`) — do not promote prototype code as-is.

**v1 scope locked with this direction:** Home, Case Studies, thin About; contact as chrome links (no Writing index); two to three Case Studies; hire-first Capsules with peer-depth below.
